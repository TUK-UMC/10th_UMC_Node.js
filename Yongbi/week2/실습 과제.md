  ### 1. 기본 조회

🔑 “소설” 카테고리의 모든 책을 조회하자.

SELECT b.name
FROM book b
JOIN book_category bc ON b.book_category_id = bc.id
WHERE bc.name= "소설";

book 테이블과 book_category를 join해서 찾음. 실습 부분에서는 description도 같이 찾았지만 나는 책 이름만 찾으면 충분하다 생각해 이렇게 쿼리문을 작성함. 

### 2. 집계 함수

이번에는 집계 함수를 사용해 보겠습니다.

🔑 회원별 대여 횟수 상위 5명을 조회하자.

SELECT m.name, COUNT(*) as rent_count
FROM member m
JOIN rent r ON m.id=r.member_id 
GROUP BY m.id, m.name
ORDER BY rent_count DESC
LIMIT 5;

COUNT(*) 라는 집계 함수를 사용함. COUNT(*) as rent_count -> 각 회원이 대여한 행의 개수 -> 대여 횟수를 rent_count라고 부르겠다는 뜻
rent 테이블과 member 테이블을 join으로 연결해서 어떤 회원이 어떤 대여 기록을 가졌는지 매칭함. 

그 후 GROUP BY m.id , m.name을 통해 회원별로 하나씩 묶어서 COUNT(*)를 셈. 

마지막에는 ORDER BY DESC를 통해 내림차순으로 정렬 후 LIMIT 5로 상위 5명 출력하는 코드 

### 3. 복합 JOIN

이제 슬슬 쿼리가 복잡해지기 시작합니다. 

🔑 각 회원이 좋아요를 누른 책의 카테고리별 분포를 조회하자.

결과 예시 ⬇️

전하경 | 소설   | 3
전하경 | 에세이 | 1

재서 | 소설   | 5
재서 | 만화   | 2

SELECT m.name, bc.name as category, COUNT(*) as like_count
FROM member m
JOIN book_likes bl ON m.id=bl.member_id
JOIN book b ON bl.book_id=b.id
JOIN book_category bc ON b.book_category_id=bc.id
GROUP BY m.id, m.name, bc.id, bc.name;


 1. 회원이 좋아요를 누른 책들을 카테고리별로 묶어서,
 2. 각 회원이 어떤 카테고리 책에 몇 번 좋아요를 눌렀는지 조회하기

SELECT 부분에서 회원 이름 , 카테고리 이름 , 좋아요 갯수 출력 ( COUNT(*) 사용 )

member 테이블에서 book_likes, book , book_category 테이블에 차례차례 JOIN 

그 후 GROUP BY를 이용해 회원별 / 카테고리별로 묶어줌. 


### 4. 서브쿼리

같은 문제를 복합 JOIN을 써서 한 번, 서브쿼리를 써서 한 번 구현해봅시다.

🔑 각 회원이 좋아요를 누른 책의 카테고리별 분포를 조회하자.

SELECT 
    m.name, 
    bc.name AS category,
    (
        SELECT COUNT(*)
        FROM book_likes bl
        JOIN book b ON bl.book_id = b.id
        WHERE bl.member_id = m.id
          AND b.book_category_id = bc.id
    ) AS like_count
FROM member m
CROSS JOIN book_category bc
WHERE (
    SELECT COUNT(*)
    FROM book_likes bl
    JOIN book b ON bl.book_id = b.id
    WHERE bl.member_id = m.id
      AND b.book_category_id = bc.id
) > 0;

이 방식은 member의 모든 회원과 카테고리를 하나씩 다 조합한다음 (CROSS JOIN)
각 조합마다 이 회원이 이 카테고리에 좋아요를 몇 번 눌렀지 ? 를 서브쿼리로 세는 방식임. 

그 후 좋아요가 >0인 경우만 남기는 쿼리문임. 

이 쿼리문의 단점은 COUNT(*)를 SELECT 에서 한번 WGERE에서 한번 총 두번 계산해서 비효율적임. 
그리고 CROSS JOIN을 사용하기 때문에 조합수가 커짐.

### Chapter 2. 실전 SQL- 어떤 Query를 작성해야 할까?

만약 이런 요구사항이 들어오면 어떻게 쿼리를 작성할 것인가? 

-> 책의 좋아요 갯수를 계산하는데, 내가 차단한 사용자의 좋아요는 집계를 하지 않는다.

차단 테이블은 다음과 같다고 가정 

select count(*)
from book_like as bl
left join block as b on bl.user_id = b.target_id and b.owner_id = 2
where bl.book_id = 3 and b.target_id is null;

book_id = 3인 책의 좋아요를 세는데, owner_id = 2가 차단한 사용자 (target_id)의 좋아요는 제외하는 쿼리문. 

책의 좋아요 갯수를 출력해야 하니 SELECT COUNT(*)
book_like 와 block을 left join한다. 
조건은 on 뒤에 있는 bl.user_id(좋아요를 누른 사람) = b.target_id(2번 사용자한테 차단당한 사람) and b.owner_id = 2 

b.target_id is null -> left join을 했을때 차단된 사용자라면 block 테이블과 매칭되서 b.target_id 값이 생기고, 
차단되지 않은 사용자라면 block쪽 값이 null이 됨. -> b. target_id is null은 차단하지 않은 사람의 좋아요만 센다 라는 뜻이 됨. 

-> UMC라는 이름을 가진 해시태그가 붙은 책을 찾기 
join 또는 subquery가 필요하겠구나 라는 생각이 듦. 나는 join을 사용하도록 하겠음. 

select b.*
from book as b
inner join book_hash_tag as bht on b.id = bht.book_id
inner join hash_tag as ht on bht.hash_tag_id = ht.id
where ht.name = 'UMC';

book 이랑 book_hash_tag랑 hash_tag랑 join 해버림. 

-> 책들의 목록을 최신순으로 조회하는 쿼리 만들기 
created_at을 이미 만들어놔서 간단하게 쿼리 작성 가능함. 

select * from book order by created_at desc;

-> 좋아요 개수 순으로 목록 조회를 하기 

select * from book as b 
join (select count(*) as like_count
				 from book_likes
						group by book_id) as likes on b.id = likes.book_id
order by likes.like_count desc;

book 자체에 likes 칼럼이 없으므로 count(*)를 사용해서 like_count 집계함.


### 페이지네이션 


페이지네이션이란 , 데이터를 한번에 다 보여주지 않고 여러 페이지로 나눠서 보여주는 것. 
이를 paging이라 하는데 이 paging에도 2가지 형태가 존재함 


## offset-paging

우리가 평소에 자주 보는 페이징으로 직접 페이지 번호를 찾아내어 이동하는 페이징이다. 

select *
from book
order by likes desc
limit 10 offset 0;

쿼리를 작성해보면 limit과 offset을 통해 데이터를 건너뛰고, 보여주는 데이터의 갯수를 정해주는 것을 알 수 있다. 
근데 이런식으로 쿼리를 작성하면 데이터가 많아질때 성능 이슈가 발생할수 있다. 
이러한 단점을 보완한 페이징이 커서 페이징이다 

## cursor-based paging

이름에서 유추 할 수 있듯이 커서로 무언가를 가르켜 페이징을 하는 방법.

SELECT * FROM post
WHERE id > 100
ORDER BY id
LIMIT 10;

limit을 사용하는 것은 offset-paging과 똑같지만 offset이 없어졌고, 
id를 사용해서 id > 100 일때 데이터를 10개 가져와라. 라는 식으로 동작하는 것을 알 수 있음.
